## How It Works

Legal documents are authored in the main (private) app repository. When changes are merged to master, a GitHub Actions workflow automatically syncs the updated documents to this repo, which triggers a GitHub Pages rebuild.

**Do not edit legal documents directly in this repo.** Changes will be overwritten on the next sync.

## Version Archive

Previous versions of legal documents are preserved in the `versions/` directory for compliance and transparency.
